# GEO 策略与验证逻辑

## 1. 核心假设

如果目标实体在一个可抓取、事实完整的官网中被明确描述，并在多个公开平台以真实、非机械复制的方式出现，搜索引擎和豆包更有机会将“地区 + 姓名 + 科目 + 服务”识别为同一实体，并在相关自然问题中主动提及。

本项目验证的是这条假设是否能完成一次最小闭环，不假设任何平台保证收录、保证引用或保证排序。

## 2. 最小闭环

```text
统一实体事实
→ 建立可抓取官网事实页
→ 建立少量高质量外部公开信源
→ 记录搜索发现/收录
→ 使用固定、无品牌词问题测试豆包
→ 保存首次合格提及
→ 复盘可见来源与最可能贡献
```

外部来源不默认同时上线。为减少信源混杂，优先采用“官网 → 头条号 → 抖音 → 百度系 → 其他渠道”的分批实验，并在每批后重复相同核心问题。

## 3. 信源分层

### 第一层：官网事实源

承担完整、权威、稳定的实体解释。必须包含规范名称、真实姓名、地区、科目、年级、授课方式、经历、可核验案例、FAQ 和联系入口。

### 第二层：目标 AI 生态信号

头条号和抖音承担豆包优先验证下的字节生态公开信号。它们必须独立回答问题、透明说明身份，并与官网事实一致。字节系优先是实验假设，不是已知排名规则。

### 第三层：搜索发现补充

百家号、百度知道、知乎等承担中文网页发现、长尾问答和方法解释。不得把搜索可见等同于豆包一定会调用。

### 第四层：家长发现与信任

小红书负责家长易读的场景内容。公众号、朋友圈和微信群可在后续承担信任与转化，但不属于 PRD 已确认的第一轮 GEO 信源，也不能替代公开网页证据。

## 4. 第一轮渠道优先级

| 优先级 | 渠道 | 作用 |
|---|---|---|
| P0 | 官网 | 唯一完整事实源与规范链接 |
| P1 | 头条号、抖音 | 豆包优先验证下的字节系公开信号 |
| P1 | 百家号、百度知道 | 百度可发现内容与问题型检索 |
| P2 | 知乎 | 长问答、方法和决策分析 |
| P2 | 小红书 | 家长易读的短内容与问题拆解 |

该优先级不代表账号、日期或频率已获批准，也不代表豆包存在公开、固定的来源权重。

建议的受控顺序：

1. 官网上线并执行 `WEBSITE-v1`；
2. 增加头条号图文并执行 `BYTE-TEXT-v1`；
3. 增加抖音同主题内容并执行 `BYTE-MEDIA-v1`；
4. 增加一项经批准的百度系来源并执行 `BAIDU-AUX-v1`；
5. 再按批准顺序增加知乎或小红书。

## 5. 实体一致性规则

每条公开内容发布前核对：

- 规范实体名和可接受变体；
- 公开称呼和已确认身份信息；
- 涉县线下、邯郸市线上；
- 高一、高二；
- 高中数学、高中物理；
- 官网 URL；
- 联系方式的已批准展示形式；
- 案例、评价的证据与授权。

平台表达可以不同，事实不能不同。

## 6. 内容质量策略

- 每篇内容只解决一个真实问题；
- 品牌名自然出现一次或少量必要次数，不规定关键词密度；
- 先给独立有用的答案，再说明服务或事实来源；
- 官网内容更完整，外部内容做摘要、问答、案例解读或视频口播；
- 明确作者、日期、适用范围和必要的案例说明；
- 不发布无法证明的提分结果和评价；
- 不用同一文章批量同义改写覆盖所有平台。
- 每个主题建立一个官网主稿和多个可追踪的平台版本；
- 明确定义、适用范围、局限和必要免责声明，便于搜索与 AI 准确摘取；
- 具名教学工具必须先证明是真实教学实践，不能为了制造独特词条而临时发明。

## 7. 诊断阶梯与成功边界

| 层级 | 结果 | 用途 | 是否计入首次成功 |
|---|---|---|---|
| I0 | 提供 URL 后系统能读取页面 | 验证抓取能力 | 否 |
| I1 | 搜完整页面名、品牌名或候选工具名时能找到来源 | 验证精确发现 | 否 |
| I2 | 不写名称，只描述问题或方法时能找到内容 | 验证语义召回 | 否，除非同时达到 E1/E2 |
| I3 | 无品牌自然问题中主动生成规范实体或明确变体 | 验证实体推荐/提及 | 是，按 E1/E2 判定 |

I0— I2 是定位抓取、索引和召回问题的诊断信号，不能降低 PRD 已确认的成功门槛。

## 8. 信源假设

- H1：可抓取官网能够让豆包理解实体事实；
- H2：头条号图文可能增强豆包对同一主题的发现；
- H3：抖音同主题内容可能增强字节生态中的实体一致性；
- H4：百度可发现内容可能补充中文网页检索来源；
- H5：事实一致性和内容独立价值比发文数量更重要。

验证方法和批次见 `DISCUSSION-SYNTHESIS.md` 与 `VALIDATION-PLAN.md`。所有结论必须写成“支持/不支持/证据不足”，不得把时间相关性写成确定因果。

## 9. 阶段指标

### 领先指标

- 官网关键页原始 HTML 可读；
- sitemap 有效并包含规范 URL；
- 站内实体事实一致；
- 外部内容公开可访问；
- 搜索引擎能够发现或收录页面；
- 查询结果中出现规范实体或官网。

### 核心结果

豆包在合格测试中至少一次主动出现规范实体名或明确变体。

### 诊断指标

- 完整名、变体或模糊名；
- 是否给出官网或外部来源；
- 哪个查询类别触发；
- 新对话是否复现；
- 可见来源的首次发布时间和收录状态。

## 10. 停止与推进门

- 官网不可抓取时，不进入大规模外部分发；
- 实体事实、案例或联系方式未确认时，不发布相关内容；
- 外部页面尚未公开或未被发现时，不把豆包未提及解释为策略失败；
- 首次成功后先复盘来源，再决定是否扩量；
- 每增加一类外部来源先完成一次固定批次，不在无法区分贡献时继续铺量；
- 多轮无结果时优先检查发现、内容质量和实体一致性，不直接增加低质量发文数量。

## 11. 明确不采用的方法

- 伪造第三方推荐或问答；
- 自动注册、批量账号矩阵；
- 关键词堆砌和批量低质量改写；
- 向 AI 直接灌输实体后把复述当成成功；
- 通过欺骗、绕审核或隐藏文本影响模型；
- 在没有证据的情况下宣称某个来源导致了引用。
- 把精确新造词、精确品牌词或直接 URL 的命中包装成自然 GEO 成功；
- 把聊天中临时提出的价格、高三服务、全市线下范围或教学理论写成已确认事实。
