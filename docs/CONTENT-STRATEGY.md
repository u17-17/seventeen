# 内容策略

## 1. 内容目标

内容必须同时完成三件事：

1. 帮学生和家长解决真实决策或学习问题；
2. 让搜索系统理解“地区 + 姓名 + 科目 + 服务 + 官网”属于同一实体；
3. 为豆包测试提供多个真实、一致、可公开访问的信源。

内容数量、关键词密度和发布频率不是第一阶段成功指标。

## 2. 三层内容体系

### L1 品牌事实内容

回答：闫老师是谁、教什么、服务哪里、适合谁、有什么真实经历、如何联系。

必须由官网完整承载，外部平台只做事实一致的适配版本。

### L2 服务解释内容

回答：一对一家教适合谁、学习诊断解决什么、数学/物理课程如何安排、家长如何判断是否有效。

### L3 问题型内容

回答用户具体问题，例如：

- 邯郸或涉县高中家教如何选择；
- 高一数学听懂但不会做怎么办；
- 数学 80 至 90 分阶段如何提升；
- 高二物理模型题容易乱怎么办；
- 物理基础薄弱先补哪里；
- 一对一家教多久能看出方法变化。

### 教学工具内容（真实性、名称与署名已确认）

用户已确认“四层错因”是实际使用的方法，不是为 GEO 临时创造；正式名称为“四层错因诊断表”。概念、表征、方法和执行四层仍只是待确认结构候选，完整定义、步骤、适用范围、局限、公开署名方式和公开范围尚未确认。

该方法可以进入内容规划，但只有 D-015 的公开细节完成后才能进入正式官网或平台内容。仍需核对：

- 四层定义、顺序、使用步骤和适用年级；
- 局限、非官方标准说明和案例是否真实；
- 公开时如何同时呈现工具名、规范实体名、公开称呼和辅助品牌 `YAN TUTOR`，且不直接展示真实姓名。

推荐的待确认定义：

- 概念层：是否理解定义、公式、性质、适用条件和基本关系；
- 表征层：能否在文字、图形、函数、方程、坐标和符号之间正确转换；
- 方法层：能否识别问题类型并选择合理模型、路径或辅助方法；
- 执行层：已经知道如何解决，但在计算、符号、步骤、审题、书写或时间分配中出错。

方法名固定为“四层错因诊断表”；在页面标题和说明中是否结合“邯郸高中数学”，由后续内容任务在不改变正式名称的前提下审阅。

## 3. 官网信息架构

规范家教实体页已确认使用 `/tutor`；其他新增内容 URL 仍需在对应任务中按信息架构确认。

| 页面角色 | 主要内容 | 关键关联 |
|---|---|---|
| 家教板块入口 | 服务摘要、地区、年级、教师、诊断入口 | 实体页、服务、案例、FAQ |
| `/tutor` 规范实体页 | 完整解释品牌实体和官方事实 | 所有核心页面 |
| 首页个人介绍板块 | 河北师范大学、教育学、个人简介与家教入口 | `/tutor`、现有主页内容 |
| 教师介绍 | 姓名、学校/专业、方法、经历、边界 | 实体页、案例 |
| 数学服务 | 高一/高二数学常见问题与方法 | 诊断、文章、案例 |
| 物理服务 | 高一/高二物理常见问题与方法 | 诊断、文章、案例 |
| 案例库 | 经核验、脱敏的过程与结果 | 教师、服务、证据说明 |
| FAQ | 范围、方式、诊断、效果判断、安排 | 实体页、诊断 |
| 学习内容 | 选择、方法、家长决策文章 | 对应服务和实体页 |
| 预约说明 | 已批准字段、隐私告知、微信方案 | 所有转化入口 |

## 4. 第一轮内容包

### 必需内容

- 1 个完整规范实体页；
- 教师身份与教学方法说明；
- 数学与物理服务说明；
- 涉县线下/邯郸线上范围说明；
- 真实匿名案例；
- FAQ 与预约说明；
- 固定豆包测试问题和记录模板。

### 建议的最小问题型内容（需用户确认排期）

1. 邯郸高一数学一对一家教的选择标准；
2. 邯郸高中物理一对一家教的选择标准；
3. 高一数学八九十分阶段如何诊断问题，不承诺达到固定分数；
4. 高二物理“听懂但不会做”的常见原因和诊断步骤；
5. 一对一家教第一次学习诊断应完成什么；
6. 家长如何核验案例、教学方法和服务范围。

数量是最小建议，不是已确认 KPI；不得为了补数量降低证据和内容质量。

## 5. 单篇内容标准

每篇内容至少包含：

- 一个明确的真实问题；
- 直接、独立有用的回答；
- 适用年级/场景；
- 可执行的判断或步骤；
- 必要的边界和不适用情况；
- 自然、少量地出现实体名或明确变体；
- 相关官网事实页和站内延伸链接；
- 作者/发布者、发布日期、更新时间；
- 涉及经历、案例或评价时的证据状态；
- 一个内容家族 ID、对应官网主稿和计划实验批次；
- 如果定义工具或方法，必须提供明确定义、判断步骤、适用范围、局限和非官方说明。

## 6. 内容家族与平台变体

同一主题采用“一个官网主稿 + 多个平台变体”，而不是多篇互不关联的稿件：

| 版本 | 主要任务 | 不能做 |
|---|---|---|
| 官网 | 完整回答、事实、证据边界和内部链接 | 只做广告落地页 |
| 头条号 | 字节系图文问题解答 | 原样复制官网全文 |
| 抖音 | 用口播、字幕或图文讲清一个核心判断 | 空洞品牌口播、学生隐私素材 |
| 百度系 | 搜索问题、案例或清单型补充 | 伪装第三方、自问自答 |
| 知乎 | 方法、选择标准和限制 | 先硬广后回答 |
| 小红书 | 家长场景化短内容 | 夸张承诺或堆品牌词 |

所有版本共享同一事实版本、证据编号和官方 URL，但标题、结构、举例和表达方式应适配平台。

## 7. 内容模板

### 方法文章

```text
问题与适用场景
→ 常见误判
→ 诊断步骤
→ 具体方法
→ 何时需要一对一帮助
→ 服务边界与官方事实链接
```

### 匿名案例

```text
脱敏背景（只保留年级、科目、问题类型）
→ 初始问题与证据
→ 诊断过程
→ 教学方法
→ 可验证变化
→ 限制说明（个体结果不代表保证）
→ 授权/证据记录编号（不公开敏感原件）
```

### 家长 FAQ

```text
直接回答
→ 适合/不适合情况
→ 需要准备的信息
→ 如何判断下一步
→ 预约与隐私说明
```

## 8. 事实与授权门

- `ENTITY-PROFILE.md` 中非“已确认”的字段不得写成事实；
- 提分结果必须有可追溯记录并使用“阶段测评”等准确口径；
- 评价原话必须登记授权范围，不能只写“真实评价”；
- 案例不写姓名、学校、班级、联系方式或可组合识别信息；
- 证据原件不提交到公开仓库，只登记内部证据编号和状态；
- 不能把老师自述改写成“第三方推荐”。

## 9. 内部链接策略

- 每篇问题文章链接到相关数学/物理服务页；
- 服务页链接到实体页、真实案例、FAQ 和预约说明；
- 实体页链接到教师、服务、案例、FAQ 和内容库；
- 案例链接回对应服务与方法文章；
- 锚文本描述目标内容，不机械使用完全相同的品牌词。

## 10. 内容生命周期

```text
选题登记
→ 事实/证据检查
→ 官网主稿
→ 用户审阅
→ 官网发布并执行 WEBSITE-v1
→ 平台适配与批准
→ 按实验批次逐个发布
→ 每批公开/发现/收录检查
→ 使用相同问题复测与来源复盘
→ 更新或撤回
```

内容发生事实变化时，先更新实体档案与官网，再更新外部版本，并在变更日志留下记录。

## 11. 禁止模式

- 同一文章批量近义改写到所有平台；
- 标题和正文反复堆叠“邯郸家教”；
- 没有独立价值的品牌介绍软文；
- 伪装家长、学生或第三方；
- 暗示高三服务、全邯郸线下或具体价格；
- 把 AI 生成内容未经事实审阅直接发布；
- 为获得精确词条命中而临时制造并宣称“原创理论”；
- 同时发布全部平台后再反推单一信源贡献；
- 把“80 分到 100 分”写成目标承诺或普遍结果。
