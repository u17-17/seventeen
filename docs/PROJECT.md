# seventeen-yan.cn 家教 GEO 首次引用验证项目

## 1. 项目摘要

本项目围绕品牌实体“邯郸闫老师高中数学物理家教”，建立统一官网事实源、可公开检索的外部内容和可复现的豆包测试流程，验证一次自然问答中的主动实体提及。

核心网站：<https://seventeen-yan.cn/>

核心目标不是批量生产内容，也不是立即建设 GEO SaaS，而是用最小、真实、可留证的闭环判断该方法是否有效。三段项目讨论中的方向与细节已按事实优先级融合，处理结论见 `DISCUSSION-SYNTHESIS.md`。

## 2. 目标用户

- 邯郸市涉县高一、高二学生及家长；
- 邯郸市范围内需要线上高中数学或物理辅导的高一、高二学生及家长；
- 需要先做学习问题诊断，再决定是否进行一对一辅导的家庭。

高三不属于当前正式服务范围。

## 3. 核心价值

### 对用户

快速理解老师是谁、服务什么学生、采用什么方法、是否适合当前问题，以及如何预约学习诊断。

### 对 GEO 验证

让官网与外部来源围绕同一组可核验事实形成一致信号，并记录 AI 是否、何时、依据哪些来源主动识别该实体。

## 4. 成功标准

第一阶段最低成功标准：豆包在一个新的独立对话中，对未包含目标实体名的自然问题，主动生成“邯郸闫老师高中数学物理家教”或可明确指向该实体的名称变体，并保存完整证据。

必须同时满足：

1. 测试问题不包含规范名称或可接受变体；
2. 同一对话中未提前介绍实体；
3. 结果不是复述用户输入；
4. 名称、地区、科目、姓名或官网事实足以排除同姓教师和通用描述；
5. 保存时间、平台、测试环境、问题、完整回答、来源链接与截图；
6. 至少一个新的独立对话出现一次合格结果。

第一阶段不要求排名第一、必须引用官网、多平台同时出现、每次稳定出现或直接带来成交。

## 5. 项目阶段

| 阶段 | 目标 | 出口条件 |
|---|---|---|
| 0 文档建设 | 建立事实、需求、审计、策略、任务与决策文档 | 用户确认审计和改造任务 |
| 1 官网审计与方案 | 只读核对代码、页面、内容和抓取状态 | 待决事项获得用户选择 |
| 2 官网开发 | 建立独立家教板块与可抓取事实源 | 技术、内容、隐私和构建验收通过 |

| 5 复盘与自动化 | 判断是否值得扩展 | 用户决定继续、调整或停止 |

## 6. 项目范围

### 包含

- 综合个人网站中的独立家教业务板块；
- 品牌实体页、教师与服务说明、真实匿名案例、FAQ、学习内容、诊断预约；
- SEO/GEO 技术基础；
- 官网、抖音、头条号、百家号、百度知道、知乎、小红书的第一轮信源；
- 发现/收录记录、豆包基线与正式验证、证据归档和来源复盘。
- 以相同核心问题集逐批增加信源的对照式实验。

### 不包含

- 完整 SaaS、多客户、收费、账号矩阵、全自动发布或注册；
- 大规模批量发文、无关关键词、自动绕过审核；
- 虚假评价、案例、第三方推荐或问答；
- 在首次闭环前开发复杂 CMS、监控后台或商业化系统。

## 7. 原则

- 真实一致优先于词频；
- 官网事实优先于平台改写；
- 小规模可验证优先于大规模自动化；
- 证据、授权和未成年人隐私优先于内容数量；
- 每个阶段必须经用户确认，不能自动扩大范围。
- 豆包信源偏好只能作为待验证假设，不能写成已知算法或保证。
- 精确品牌词、精确新造词或直接提供 URL 的命中只用于技术诊断，不计入首次成功。

## 8. 讨论融合后的执行主线

```text
BASELINE-v1
→ 官网事实源和最小内容包
→ WEBSITE-v1
→ 头条号图文 + BYTE-TEXT-v1
→ 抖音同主题内容 + BYTE-MEDIA-v1
→ 百度系补充 + BAIDU-AUX-v1
→ 其他已批准来源 + OTHER-SOURCES-v1
→ 首次 E1/E2 后立即复盘
```

这是建议的实验顺序，不是发布授权。实际账号、时间、频率和是否继续到下一批，仍按 `DECISIONS.md` 执行。
