# 咨询表单与隐私方案

状态：用户已确认方案 A、90 天留存、“升高一咨询”、删除“其他”年级选项、隐私权利联系渠道、限流优先使用平台能力，以及通知邮箱独立标签和每周清理方式。T-206 本地代码改造与验证已完成，尚未发布；平台级限流、邮箱标签和 90 天清理流程仍需在发布/运营验收时核对。

## 1. 当前实现

T-206 完成后的本地流程为：

```text
浏览器咨询表单
→ POST /api/diagnosis
→ Vercel Serverless Function 校验和组装邮件
→ Resend
→ 私人通知邮箱
```

当前没有独立数据库。表单只收集已批准的必填联系信息、年级、科目、授课偏好和主要问题，以及可选联系人称呼、成绩区间和沟通时间；不再收集精确分数或目标分数。页面直接展示完整隐私告知并要求未预先勾选的明确同意。

本地实现已加入前后端共享允许值和长度限制、8 KB 请求体限制、来源校验、真实 honeypot、最短填写时间、请求 ID、Resend 幂等键、邮件 HTML 转义、发送错误检查以及微信/二维码/可复制摘要回退。应用日志不记录联系方式或问题正文。平台级速率限制不由本地 Serverless 实例内计数器替代，需在实际 Vercel 发布配置中核对。

## 2. 三种可选方案

| 方案 | 数据路径 | 优点 | 代价/风险 | 建议 |
|---|---|---|---|---|
| A. 最小邮件通知 | 表单 → API → Resend → 通知邮箱；不建数据库 | 保留现有能力、改动小、数据面最小 | 需管理邮箱留存；查询和统计能力弱 | **第一阶段推荐** |
| B. 邮件 + 轻量数据库 | 表单 → API → 数据库 + Resend | 可跟进状态、统计来源和防重复 | 增加数据库、权限、删除流程和泄露面 | 首次闭环后再评估 |
| C. 仅微信联系 | 不提交表单，只展示微信号和二维码 | 隐私和接口风险最低 | 失去结构化诊断、邮件提醒和来源统计 | 可作为故障回退，不建议取代主方案 |

## 3. 推荐方案 A：最小邮件通知

### 3.1 推荐字段

| 字段 | 是否必填 | 推荐形式 | 原因 |
|---|---|---|---|
| 联系人称呼 | 否 | 最多 30 字 | 回复时称呼方便，但不是完成咨询所必需 |
| 家长微信号或手机号 | 是 | 单一联系字段，5—50 字 | 用于回复咨询；不收集学生个人联系方式 |
| 学生年级 | 是 | 高一 / 高二 / 升高一咨询 | “升高一咨询”仅为咨询入口，不扩展正式长期服务年级；删除“其他” |
| 咨询科目 | 是 | 数学 / 物理 / 数学和物理 / 暂不确定 | 服务匹配 |
| 授课偏好 | 是 | 涉县线下 / 邯郸线上 / 暂不确定 | 直接落实服务区域边界，不需要详细住址 |
| 当前成绩区间 | 否 | 未提供 / 60 以下 / 60—79 / 80—99 / 100—119 / 120 及以上 | 比精确分数更符合最小化原则，也足够用于初步判断 |
| 主要问题 | 是 | 8—500 字 | 提供初步诊断所需信息 |
| 可沟通时间 | 否 | 固定选项 + 需要再沟通 | 可在首次回复时再确认 |
| 隐私同意 | 是 | 未预先勾选的复选框 | 记录知情和明确选择 |

建议删除“目标分数”。它不是回复咨询所必需，还容易被误解为网站对结果的承诺。

表单不得要求或鼓励填写：学生姓名、学校、班级、身份证号、学生手机号、详细家庭地址、试卷照片或聊天截图。

### 3.2 推荐隐私告知

页面在提交按钮前以可直接阅读的方式显示摘要，并链接到完整说明：

> 本表单由闫老师用于回复高中数学、物理家教咨询和进行初步学习问题判断。我们会收集联系人称呼、联系方式、学生年级、咨询科目、授课偏好、可选成绩区间、主要问题和可选沟通时间。信息将通过 Resend 发送至私人通知邮箱，不用于公开展示、广告群发或出售。通知邮件会在最后一次咨询沟通后 90 天内删除；你可通过微信号 `-L09-29` 请求查阅、更正、删除或撤回同意。请由家长或监护人提交，不要填写学生姓名、学校、班级、详细地址等可识别信息。

推荐复选框：

> 我已阅读并同意《咨询表单个人信息说明》，确认我是学生家长/监护人，或已经取得监护人同意。

《中华人民共和国个人信息保护法》要求处理目的明确、收集限于最小范围，并在处理前以清晰方式告知处理者、目的、方式、信息种类、保存期限和权利行使方式；基于同意处理时，同意应在充分知情前提下自愿、明确作出。参考：[中国人大网法律全文](https://www.npc.gov.cn/npc/c2/c30834/202108/t20210820_313088.html)。本文是产品实施方案，不替代针对具体业务的法律意见。

### 3.3 推荐留存规则

- 不建立业务数据库；
- Resend 只承担事务通知，不用于营销邮件或受众列表；
- 通知邮箱设置独立标签，并每周清理最后一次咨询沟通超过 90 天的记录；
- 如果后续建立正式授课关系，需要保留的课程记录另行告知，不沿用本表单的无限期留存；
- 不在应用日志、分析事件或错误信息中记录联系方式和问题全文；
- 发布前核对 Resend 的服务条款、数据处理位置和适用的额外告知要求。

用户已确认留存期限为 90 天；发布/运营验收前必须核对与该期限一致的邮箱标签和实际清理方式。

### 3.4 API 与反滥用要求

第一层，不增加新数据库即可完成：

- 只接受 `POST` 和 `application/json`；
- 请求体上限建议 8 KB；
- 年级、科目、授课偏好和成绩区间使用服务端允许值列表；
- 所有字符串设置长度上限并转义邮件 HTML；
- honeypot 的真实值必须进入请求体；
- 增加最短填写时间判断，拦截瞬时机器人提交；
- 校验 `Origin`/`Host`，只接受正式站点和本地开发来源；
- 每次提交生成请求 ID，并使用 Resend 幂等键避免网络重试重复发信；
- 显式检查 Resend 返回的 `error`，失败时返回失败并保留微信回退；
- 缺少 `RESEND_API_KEY`、`FROM_EMAIL` 或 `NOTIFY_EMAIL` 时明确失败；
- 日志只记录请求 ID、时间、来源页面和结果，不记录联系方式与问题正文。

第二层优先使用 Vercel 平台级速率限制，暂不增加带短期过期时间的共享限流存储；如后续需要存储型限流，需另行确认。

无论选择哪种，都不使用 Serverless Function 内存计数器作为正式限流，因为不同实例之间无法可靠共享状态。T-206 未记录 IP 或哈希 IP；平台级限流在实际发布配置中核对。

### 3.5 成功与失败体验

- 成功：明确说明“已收到咨询，会按所填联系方式联系”，但不承诺具体回复时间；
- 失败：显示已确认的微信号与二维码，允许复制已经填写的咨询摘要；
- 重复提交：提示已经收到，不重复发信；
- 用户不同意：仍可直接复制微信号联系，但表单不能提交；
- 页面不得把“生成咨询信息”和“已经发送成功”混为同一状态。

## 4. 方案 B：邮件加轻量数据库

仅在需要管理大量咨询时考虑。最小表结构可包含：随机 ID、创建时间、来源页面、年级、科目、授课偏好、成绩区间、问题摘要、联系状态和删除时间。

实施前必须额外解决：数据库地区、访问权限、加密、备份、管理员登录、导出/删除请求、保留期限和安全事件处理。第一轮 GEO 验证不需要这些能力，因此当前不推荐。

## 5. 方案 C：仅微信联系

保留微信号、二维码和一段可复制的咨询模板，不向服务器提交任何内容。适合作为 Resend 故障时的回退，也可在用户不愿意同意表单处理时使用。
