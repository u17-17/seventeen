## 2. 内容需求

| ID | 规则 |
|---|---|
| CR-001 | 规范实体名为“邯郸闫老师高中数学物理家教”。 |
| CR-002 | 可接受变体必须结合地区、科目、姓名、官网或高度一致事实明确指向本实体。 |
| CR-003 | 公开网页不直接展示真实姓名，公开称呼使用“闫老师”；学校为河北师范大学，专业为教育学。 |
| CR-004 | 不得出现具体课时价格或数字报价。 |
| CR-005 | 不得把高三写入主要服务范围，不得创建“高三家教”服务页。 |
| CR-006 | 不得描述为“邯郸全市均可线下授课”。 |
| CR-007 | 案例必须真实，学生身份必须脱敏。 |
| CR-008 | 家长或学生评价必须真实；需要授权的原话在授权前不得发布。 |
| CR-009 | 不得伪造第三方推荐、百度知道回答或其他平台身份。 |
| CR-010 | 外部平台允许风格适配，但核心事实必须与官网一致。 |
| CR-011 | 官网重要事实以可检索文本呈现，不能只存在于图片。 |
| CR-012 | 综合个人网站定位必须保留，家教业务不能未经批准取代全部个人内容。 |
| CR-013 | 平台内容必须关联到一个官网主稿或官方事实页，核心事实一致但表达不能机械复制。 |
| CR-014 | 任何具名教学方法或“原创工具”只有在用户确认真实使用、定义和公开意愿后才能发布。 |
| CR-015 | 精确品牌名、精确新造词或直接提供 URL 的测试只可用于技术诊断，不计入首次 GEO 成功。 |
| CR-016 | 外部来源应按批准的实验批次逐步增加，并记录新增来源与测试结果，避免一次性铺开后无法复盘。 |

## 3. 表单与联系需求

PRD 建议收集年级、地区、科目、成绩区间、主要问题、线上/线下偏好和联系方式。现有实现与三种可选方案的完整比较见 `FORM-PRIVACY-OPTIONS.md`。

已确认保留 Resend 邮件接口。实施前仍必须确认：

- 当前微信号和二维码具体值是否仍有效；
- 收集字段、必填项和数据保留期限；
- 隐私告知、同意机制和删除/联系渠道；
- 反滥用、限流和失败回退。

当前推荐方案是保留 Resend 的最小邮件通知、不建业务数据库、删除目标分数、精确分数改为可选区间，并增加授课偏好、显著隐私告知和未预选同意框。该推荐尚未自动升级为已确认需求。

在确认前不得把当前源码中的既有实现视作最终需求。

## 6. 质量边界

### 始终执行

- 先核对事实和授权，再写内容；
- 实施任务必须有验收与验证命令；
- 逻辑修改先补测试；
- 发布和测试结果必须登记。

### 必须先问

- URL、联系方式、表单与存储；
- 技术栈和部署变更；
- 首页和原个人网站板块；
- 外部账号、频率和发布时间；
- 任何新事实、案例、评价或个人信息。

### 永不执行

- 提交密钥或未脱敏个人信息；
- 伪造信源、评价、案例或 AI 结果；
- 删除失败测试来让构建通过；
- 未经批准扩大到自动化、SaaS 或批量账号矩阵。

## 7. 需求完成定义

第一阶段完成需同时满足 PRD 最终验收清单 12 项。单独完成页面开发、外部发文或一次含品牌词的问答都不能宣告项目完成。
