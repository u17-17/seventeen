# 现有网站只读审计

审计日期：2026-07-15（Asia/Shanghai）  
范围：本地仓库 `main` / `86a8bf4`、线上 <https://seventeen-yan.cn/>、公开 HTTP 响应、现有自动化测试。  
限制：未登录 Vercel、搜索站长平台或外部内容账号；未提交真实表单；搜索结果抽查不等同于各搜索引擎的完整收录证明。

## 1. 结论

现有站点具备成熟的家教展示界面、多个独立客户端路由、表单/API 和基础客户端 SEO 逻辑，但尚不能承担 PRD 要求的 GEO 官方事实源角色。

最重要的原因是：目标实体及其关键事实没有完整进入站点；正文和页面级元数据依赖 JavaScript；线上 sitemap 无效；未知路径返回 200；服务地区表达与 PRD 不一致。应先完成事实、抓取和 HTTP 基础，再建设外部信源。

## 3. GEO/SEO 阻塞项

### [P0] 规范实体和关键事实缺失

- 位置：`src/data/siteData.js`、`src/data/pageData.js`、`src/utils/seo.js`、`index.html`
- 现状：主要品牌是 `YAN TUTOR`；未出现完整规范实体名和真实姓名字段；涉县线下边界、学校与专业也未形成完整事实。
- 影响：搜索系统难以把品牌、姓名、地区和科目稳定合并为目标实体。
- 建议：先核验实体档案，再从统一事实源生成实体页、SEO 和结构化数据。

### [P0] 核心正文和页面 SEO 依赖 JavaScript

- 位置：`index.html`、`src/components/Seo.jsx`、`src/utils/seo.js`、当前 Vite SPA 输出
- 现状：原始 HTML 只有空根节点；独立页标题、描述、canonical、JSON-LD 在 `useEffect` 后注入。
- 影响：不执行或延迟执行 JavaScript 的搜索/AI 抓取器只能看到通用壳，无法读取完整事实；分享爬虫也可能获得错误元数据。
- 建议：对所有关键路由做静态生成/预渲染或等价服务器输出，并以原始 HTTP HTML 验收。

### [P0] sitemap 无效

- 位置：线上 `/sitemap.xml`、`scripts/generate-sitemap.mjs`、`public/`
- 现状：本地 `public/sitemap.xml` 不存在；线上路径被 SPA rewrite 到首页 HTML。
- 影响：搜索系统没有可靠 URL 清单；错误的 200 响应可能掩盖部署遗漏。
- 建议：构建阶段生成 XML，部署后验证状态、Content-Type、正文和 URL 数量；在 robots 中声明地址。

### [P0] 服务区域表述不符合 PRD

- 位置：`src/data/siteData.js`、`src/data/pageData.js`、`src/utils/seo.js`
- 现状：多处使用“线上线下均可”“河北邯郸及线上”，没有明确限制涉县线下。
- 影响：官网事实与已确认服务范围冲突，后续外部内容会复制错误信号，也可能误导用户。
- 建议：采用实体档案的统一地区表达，并做全站事实一致性测试。

## 4. 主要风险项

### [P1] 综合个人网站定位与现状冲突

当前源码和线上页面几乎全部是家教品牌站，未发现 PRD 所称的其他个人网站板块。需要确认这些内容是否在其他分支/仓库，或需要重新引入；在确认前不能把“保留原内容”转化为可执行改造。

### [P1] 未知路径返回 200

`vercel.json` 将所有非 API 路径重写到首页，客户端对未知路径也回落到首页。会产生 soft 404、重复页面和抓取质量问题。需要真实 404 路由与部署规则。

### [P1] 案例与评价缺少仓库内证据状态

站点把三条评价描述为“真实”，并公开“阶段测评提升约 20 分”等结果；仓库没有授权/证据台账。不能据此断言内容不真实，但在扩散到 GEO 外部信源前必须逐条核验和登记。

### [P1] 表单收集个人信息但缺少告知与同意

表单收集家长称呼、手机号或微信号、学生年级、成绩和学习问题，页面未见隐私用途、保留期限、联系/删除方式或明确同意。PRD 又把字段和存储列为待确认，实施前需要完成隐私与数据最小化决策。

### [P1] 公开邮件接口缺少强反滥用控制

API 只有 honeypot，没有可见的速率限制、请求体总大小限制、来源校验或监控。可能造成垃圾请求、邮件额度消耗和通知淹没。是否保留接口需先确认。

### [P1] README 自相矛盾

README 前部称“当前预约流程不接入后端”，后部又说明 `/api/diagnosis` 和 Resend；源码确实会 POST。会误导维护者和测试流程。

### [P1] 教师头像资源缺失

`AboutSection.jsx` 引用 `/teacher-avatar.jpg`，但 `public/` 没有该文件；线上该路径因 rewrite 返回 HTML。浏览器最终回退占位，但每次产生一次无效请求，也掩盖真实资源 404。

### [P1] 现有年龄与身份信息存在时效/完整性风险

Hero 中硬编码“22 岁”，该事实不在 PRD 实体档案内，会随时间失效；学校、专业和真实姓名反而缺失。应明确哪些身份字段长期公开以及如何维护。

## 5. 次要改进项

### [P2] 结构化数据过于通用且只在客户端注入

当前类型为 `EducationalOrganization`，名称为 `YAN TUTOR`，地区是“河北邯郸及线上”。需要在实体/业务模型确认后选择更贴合的 Person/服务/组织组合，并在原始 HTML 输出。

### [P2] Open Graph 基础 HTML 不完整

初始 HTML 缺少 canonical 和 `og:url`，图片为相对 SVG；独立页在分享爬虫不执行 JS 时都会得到首页标题。建议输出每页静态且绝对的 OG 字段，并核对平台兼容图片格式。

### [P2] 自动化测试没有覆盖部署行为

21/21 单元测试通过，但“crawlable”测试只断言路由字符串存在，未验证原始 HTML 和线上 HTTP。需要部署级验收测试。

### [P2] 内容与实体事实分散

品牌、地区、微信、API 错误提示、SEO 和 README 各自硬编码，容易漂移。应建立单一实体事实数据源，但具体数据模型待实施方案确认。

## 6. 前端技术质量评分

该评分依据界面审计技能的技术维度，独立于上面的 GEO 就绪度。

| 维度 | 分数 | 关键发现 |
|---|---:|---|

| Responsive | 2/4 | 已有移动断点和 44px 触控目标；Hero 大标题使用 `whitespace-nowrap`，在 320px 窄屏存在裁切风险 |

| Anti-patterns | 2/4 | 溶解 Hero 有辨识度，但大量同质卡片、圆形图标、默认 Inter、装饰性 glass/blur 和侧边色条呈现明显模板化痕迹 |

### Anti-pattern verdict

未达到“完全像通用 AI 落地页”的程度，因为首屏溶解动效和品牌色有明确特征；但同质卡片网格、圆角卡片套卡片、图标加标题加正文的重复结构、Inter 默认字体和 `border-l-2` 装饰条仍有明显生成式模板痕迹。后续如改版，应先整合信息架构与证据表达，不应只做视觉抛光。
