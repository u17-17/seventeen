# 外部平台内容规范

状态：内容适配规则。账号、发布时间、频率和实际发布均待用户确认。

## 1. 通用规则

- 每个平台内容必须独立解决问题，不能只为放品牌词而存在；
- 核心事实以 `ENTITY-PROFILE.md` 为准；
- 品牌名自然出现，禁止固定词频和机械重复；
- 允许平台语气、长度、标题和素材形式不同；
- 案例、评价、图片、板书和对话截图必须通过证据、授权与脱敏门；
- 不公开具体价格，不宣传高三服务，不暗示邯郸全市线下；
- 不冒充第三方、家长或学生；
- 发布后立即登记到 `CONTENT-SOURCES.md`。

## 2. 平台矩阵

| 平台 | 实验/业务角色 | 合适内容 | 推荐结构 | 特别风险 |
|---|---|---|---|---|
| 官网 | P0 官方主信源 | 完整实体、服务、案例、FAQ、长文 | 权威事实 + 深度解释 + 内链 | 客户端不可抓取、事实漂移 |
| 头条号 | P1 字节图文实验 | 本地教育问题、完整方法文章 | 问题 → 判断 → 方法 → 边界 → 官网 | 与官网整篇复制、营销感过重 |
| 抖音 | P1 字节多形态实验 | 口播、知识点、问题诊断、本地服务说明 | 问题 → 单一方法 → 适用人群 → 官方身份 | 夸张标题、未授权学生素材 |
| 百家号 | P1 百度发现补充 | 家教选择、学习方法、问答型文章 | 搜索问题 → 直接答案 → 清单 → 事实来源 | 关键词堆砌、低质量同义稿 |
| 百度知道 | P1 百度问题补充 | 真实问题回答 | 先直接回答，再给判断标准 | 伪装第三方、自问自答造假 |
| 知乎 | P2 长尾问答 | 经验分析、选择建议、长问答 | 观点 → 原因 → 案例/方法 → 局限 | 软文、无证据结果承诺 |
| 小红书 | P2 家长发现 | 家长易读清单、学习问题拆解 | 场景标题 → 3–5 点 → 适合/不适合 → 联系边界 | 过度营销、未成年人照片/成绩单 |

建议顺序为官网、头条号、抖音、百度系、知乎/小红书；每新增一类来源后先完成公开/发现检查和固定豆包测试。该顺序仍受 D-010 的账号与发布时间批准约束。

## 3. 官网

### 目标

成为最完整、最稳定的实体事实源，而不是平台文章的复制仓库。

### 必须

- 原始 HTML 可读；
- 清晰显示规范实体名和真实姓名；
- 明确涉县线下、邯郸线上、高一高二；
- 提供更新时间、内部链接和适当结构化数据；
- 案例/评价可追溯。

## 4. 抖音

### 可用选题

- 高一数学听懂但不会做的三个常见原因；
- 高二物理综合题先判断哪个模型；
- 家长第一次找一对一家教应该问什么；
- 涉县线下辅导与邯郸线上辅导如何选择。

### 不做

- 未授权学生出镜、成绩单或聊天记录；
- “几节课必提多少分”等承诺；
- 只读品牌广告的空洞口播。

## 5. 头条号与百家号

- 同一选题必须分别改写角度和结构，而非换标题复制；
- 标题描述用户问题，不用夸张数字和保证；
- 正文先提供完整答案，品牌信息作为真实作者/服务背景；
- 官网链接是否可直接放置按平台规则执行，但来源台账必须保留。

头条号优先承担 `BYTE-TEXT-v1`，百家号优先承担 `BAIDU-AUX-v1`。同一实验批次不得为了凑数量同时发布多个未登记版本。

## 6. 百度知道

- 只回答真实存在、自己有能力回答的问题；
- 如果回答者与服务提供者是同一人，必须避免伪装中立第三方；
- 不组织虚假提问、虚假采纳或多账号互证；
- 不把“推荐闫老师”作为每个问题的固定结尾。

## 7. 知乎

- 强调可迁移的方法、选择标准和限制；
- 对个人经历使用第一人称并说明适用范围；
- 匿名案例写过程，不用未经证明的戏剧化结果；
- 回答要在没有品牌段落时仍具独立价值。

## 8. 小红书

- 使用短段落、清单和家长易懂语言；
- 封面文字只呈现问题和范围，不堆叠品牌词；
- 图片优先使用自制知识图、经授权板书或不含身份信息的素材；
- 不展示未成年人的可识别个人信息。

## 9. 第一轮范围之外的渠道

- 公众号可用于后续长文沉淀和信任，但不自动加入第一轮 GEO 来源；
- 朋友圈、微信群用于真实本地转化，不属于公开可抓取信源；
- 地图或本地商家平台只有在存在真实固定场所、平台类目允许且用户批准时才考虑；
- 不为地图入驻伪造地址，也不把邯郸市线上范围写成全市线下服务。
