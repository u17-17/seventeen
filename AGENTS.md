## 6. 命令

在进入开发阶段后使用以下命令；阶段 0 只运行不会产生持久构建产物的检查。

```powershell
npm install
npm run dev
npm test
npm run build
$env:SITE_URL = "https://seventeen-yan.cn"; npm run sitemap
```

- 每个逻辑或行为修改必须先按测试驱动流程补充或更新测试，再运行 `npm test`。
- 每个开发任务结束至少运行 `npm test` 和 `npm run build`。
- SEO/路由任务还必须用真实 HTTP 请求验证状态码、Content-Type、canonical、sitemap、robots 和 404，不能只依赖单元测试。

## 7. 实施规则

- 一次只执行 `docs/TASKS.md` 中一个已获批准的任务，控制在约 5 个文件内；超出时先拆分任务。
- 保留现有 React + Vite 技术栈和现有主页；关键路由采用构建时静态输出/预渲染。任何框架迁移或首页大改必须重新批准。
- 关键事实必须从单一实体配置或内容数据源生成，避免页面之间手工复制后发生漂移。
- 重要页面必须具备独立 URL，并在未执行 JavaScript 时仍能读取核心正文和关键元数据。
- 每个新增页面必须提供唯一标题、描述、canonical、合理标题层级、内部链接、图片替代文本、Open Graph 和适用的结构化数据。
- 不将关键词密度、批量同义改写或 AI 生成数量作为完成标准。
- 不提交密钥、邮箱、未脱敏学生资料或授权证明原件。
